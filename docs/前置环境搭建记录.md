## 6. WSL2 / Ubuntu 安装记录
### 6.1 WSL 安装
安装命令：

```powershell
wsl --install -d Ubuntu
```

过程中曾遇到：

```plain
服务器返回无效或不可识别的响应
错误代码: Wsl/InstallDistro/0x80072f78
```

原因：

```plain
WSL 主程序安装成功，但 Ubuntu 发行版下载失败，通常与 Microsoft Store 或网络下载源有关。
```

解决思路：

```powershell
wsl --update
wsl --shutdown
wsl --list --online
wsl --install Ubuntu-24.04 --web-download
```

如果 Ubuntu 24.04 下载失败，可尝试：

```powershell
wsl --install Ubuntu-22.04 --web-download
```

### 6.2 Ubuntu 初始化
进入 Ubuntu 后更新系统：

```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y curl wget unzip vim net-tools
```

### 6.3 常见问题
| 问题 | 原因 | 解决方式 |
| --- | --- | --- |
| Ubuntu 下载失败 | Microsoft Store / 下载源网络问题 | 使用 `--web-download` |
| 发行版安装异常 | 半安装状态残留 | `wsl --unregister Ubuntu`<br/> 后重装 |
| Docker 在 Ubuntu 里不可用 | Docker Desktop 未开启 WSL 集成 | Docker Desktop 设置中启用对应 Ubuntu 集成 |

### 11.5 实际遇到的问题一：控制台打不开
现象：

```plain
浏览器无法打开 http://127.0.0.1:18080
```

排查命令：

```powershell
docker ps -a
docker logs apihub-nacos --tail=100
Test-NetConnection 127.0.0.1 -Port 18080
```

原因之一：

```plain
容器显示 Up Less than a second，说明 Nacos 可能在反复重启。
```

解决思路：

```plain
先看日志，不要只看浏览器。
如果日志没有出现启动成功信息，优先处理容器启动失败。
```

### 11.6 实际遇到的问题二：NACOS_AUTH_TOKEN 不是 Base64
报错：

```plain
env NACOS_AUTH_TOKEN must be set with Base64 String.
```

原因：

```plain
Nacos 3.0.3 启动脚本会校验 NACOS_AUTH_TOKEN。
即使 NACOS_AUTH_ENABLE 设置为 false，仍可能因为环境变量残留或脚本校验导致启动失败。
```

最终解决：

```plain
保留 NACOS_AUTH_ENABLE=false
同时给 NACOS_AUTH_TOKEN 配置一个合法 Base64 字符串。
```

使用的 Base64 Token：

```plain
YXBpaHViLWFnZW50LW5hY29zLWF1dGgtdG9rZW4tMjAyNjA2MTgtdmVyeS1sb25nLXNlY3JldA==
```

强制重建：

```powershell
docker rm -f apihub-nacos
Remove-Item -Recurse -Force .\data\nacos
docker compose up -d --force-recreate nacos
```

检查最终生效配置：

```powershell
docker compose config | findstr /I "NACOS_AUTH"
docker inspect apihub-nacos --format "{{range .Config.Env}}{{println .}}{{end}}" | findstr /I "NACOS_AUTH"
```

### 14.5 PowerShell 脚本语法问题
曾遇到：

```plain
表达式或语句中包含意外的标记“)”
表达式或语句中包含意外的标记“}”
不允许使用空管道元素
```

原因：

```plain
PowerShell 中用哈希表拼复杂 JSON 时，括号或数组结构容易写错。
```

解决：

```plain
改用 Here-String 原始 JSON 字符串。
```

示例：

```powershell
$body = @"
{
  "model": "qwen-plus",
  "messages": [
    {
      "role": "user",
      "content": "Please answer in one sentence: who are you?"
    }
  ],
  "temperature": 0.3,
  "max_tokens": 200
}
"@
```

### 14.6 PowerShell 中文乱码问题
曾出现乱码：

```plain
.env 鏂囦欢涓嶅瓨鍦...
```

实际含义：

```plain
.env 文件不存在，请确认当前目录是 D:\apihub-agent-dev
```

原因：

```plain
PowerShell 控制台编码与脚本输出编码不一致。
```

解决方式：

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

或者测试脚本中尽量输出英文提示，避免中文编码问题。

### 14.7 `.env` 路径问题

原因：

脚本和 .env 不在同一目录，或者脚本按相对路径读取 .env。

## 18. 已解决问题汇总
| 问题 | 现象 | 原因 | 解决方式 |
| --- | --- | --- | --- |
| npm 无法运行 | `npm.ps1`<br/> 被禁止 | PowerShell 执行策略限制 | `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned` |
| WSL 安装 Ubuntu 失败 | `0x80072f78` | Ubuntu 发行版下载失败 | 使用 `--web-download`<br/> 或换 Ubuntu 版本 |
| MySQL 无法连接 | `Can't connect to MySQL server` | 第一次启动仍在初始化 temporary server | 等待日志出现 `port: 3306` |
| MySQL socket 报错 | `mysqld.sock`<br/> 不存在 | 连接方式或初始化阶段问题 | 等待正式启动，或使用容器内命令验证 |
| Redis 连接验证 | 需要确认是否启动 | 无 | `docker exec -it apihub-redis redis-cli ping`<br/> 返回 `PONG` |
| Nacos 控制台打不开 | 浏览器无法访问 `18080` | 容器未真正启动或反复重启 | 查 `docker ps -a`<br/> 和 `docker logs` |
| Nacos Token 报错 | `NACOS_AUTH_TOKEN must be set with Base64 String` | Nacos 3.0.3 启动脚本要求 Base64 Token | 配置合法 Base64 Token |
| Nacos 内存偏大 | 默认 `-Xms1g -Xmx1g` | Docker Desktop 本地资源有限 | 设置 `JVM_XMS=256m`<br/>、`JVM_XMX=512m` |
| DashScope 脚本语法错误 | PowerShell 括号解析失败 | 哈希表拼 JSON 写法易错 | 改用 Here-String JSON |
| PowerShell 中文乱码 | 输出中文变成乱码 | 控制台编码不一致 | 设置 UTF-8 或输出英文 |
| `.env`<br/> 找不到 | 脚本提示文件不存在 | 脚本路径和 `.env`<br/> 路径不一致 | 使用绝对路径或放到同一目录 |
| Nginx `/api/`<br/> 不可用 | 访问 API 失败 | 后端 Gateway 尚未启动 | 当前正常，后端开发后再联调 |
